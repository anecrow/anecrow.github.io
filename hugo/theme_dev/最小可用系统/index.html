使用基本知识构建最小可用的模板系统.

通用外壳模板 Baseof1

将通用的结构抽象成公共的基础模板作为外壳.

layouts/ 下的 *baseof.html 可以看作是一种预留可更改组件的外壳
各页面通用的 <doctype> <html> <head> 等固有标签都可以预先设定
之后在各个页面仅需要对预留的 block 区域进行定义替换

新建主题时在 _default/ 下默认生成了一个写好的基础骨架 baseof.html :

<!DOCTYPE html>
<html>
    {{- partial "head.html" . -}}
    <body>
        {{- partial "header.html" . -}}
        <div id="content">
            {{- block "main" . }}{{- end }}
        </div>
        {{- partial "footer.html" . -}}
    </body>
</html>

结构很经典, 看起来暂时没有需要改动的地方

  • partial 语句可以认为是一种变动不大的固有小组件, 即插即用
    • 语法为 {{ partial "<name>.html" }}, <name>.html 则对应了 layouts/partrials/ 目录下对应名称的html组件2
    • partial 后跟 . 则意为传递当前页面上下文, 有的小组件需要使用, 比如 head.html 中需要获得当前页面的 title 值来设置页面标题3
    • 两边的 - 会让模板处理时清除两侧空白行4
      • 用在这只会将最后生成的代码缩进弄得一团糟, 但没人需要维护它所以没有关系
  • block 语句开头到 end 语句结束, 定义了一块可替换的区域
    • 语法为 {{ block "<name>" }}{{ end }}, <name> 是块名
    • block 后跟 . 传递上下文, 语句中间允许有内容且包含在传递值中
    • 在对应 template 页使用 {{ define "name" }}{{ end }} 定义同块名的区域

小组件 Partial5

partial 可以认为是一种变动不大的固有小组件, 即插即用

简单使用 partial 组件的功能来对 <head> 部分进行定义 修改 layouts/partrials/ 目录下的 head.html 文件

<head>
    <title>{{ .Title }}</title>
</head>

意为使用 {{ .Title }} 语句获取当前页面的 title 属性来填充 <title> 标签, 以改变标签页所显示的文字

不同的页面拥有不同的 title 属性值, 比如:

  • 首页显示的是根目录 config.toml 中定义的 title 值
    • 这里是 My New Hugo Site
  • 而聚合页面显示的是 section 名的复数形式
    • 比如有个目录 content/post/ 地址为 localhost:1313/post/ 那么它的标题则为 Posts
  • 文章页的标题则由文章 frontmatter 中的 title 值所定义

任何页面有对应的 frontmatter 时都会直接使用该 title 属性

同样可以在 header.html 和 footer.html 中定义对应内容: Header Footer Style

页面模板 Template

页面模板套用遵循 Lookup Order 规则

hugo 首先会将页面分为几种主要类型: Kind
确定页面类型后寻找类型对应的模板文件: Template
而找到模板文件之后按照内容*决定是否继续寻找对应的壳模板文件 Baseof

为了能在同一个 baseof 壳模板基础上让不同页面模板分别填充 block 内容块, 对应类型的模板页必须只有对 block 的实现, html内容不能写在 define 块外

官方也有对该现象的说明: Override the Base Template

情况显然有些复杂, 简单举例:

假设存在 _defualt/baseof.html 且有着 {{ block "main" }} 预定义语句

  1. 缺失对应类型模板文件 (比如访问首页, 在 layouts/ 中却没有定义 index.html 或 list.html)
    • 无法渲染, 控制台输出 WARN 报错: found no layout file for "HTML" for kind "home": You should create a template file which matches Hugo Layouts Lookup Rules for this combination.
  2. 存在对应类型模板文件, 但html内容写在 define 块外
    • 使用本模板进行全部渲染, 但会忽略对应的 baseof 壳模板

    • 内容为空的模板文件也属于这个情况

  3. 存在对应类型模板文件, 仅对对应 baseof 壳模板中预留的 block 进行实现
    • 壳模板应用正常, 本模板应用正常

所以为了让前面所做的工作正常运作我们需要对 layouts/_default/ 下的 list.html 和 single.html 添加 block main 相应实现

简单做个空定义就行

{{ define "main" }}
    
{{ end }}

不打算使用独特首页模板的话可以删除 layouts/index.html

运行后就能看到对应页面的标题

聚合页面 List6

在外壳模板的基础上定义 List页

主页, section页, tags页 都是聚合页面, 它们的样式应用模板最底层的指向为 _default/list.html7

List页 一般作为导航页, 容纳对应目录的下属页面

参照官方示例改过来用:

{{ define "main" }}
    <ol>
        {{ range .Pages }}
            <li>
                <a href="{{.Permalink}}">{{.Title}}</a>
                <p>{{ .Summary }}</p>
            </li>
        {{ end }}
    </ol>
{{ end }}
  • 语句块 {{ range $array }}{{ end }} 提供了迭代方法, 每次迭代都会将语句中的内容添加进正文8
    • 语句块内部的 . 指向会被重定向至迭代中的当前项
    • .Pages 获取的子项默认排序方式 Weight > Date > LinkTitle > FilePath9
  • {{ .Permalink }} 绝对链接10
  • {{ .Title }} 页面标题10
  • {{ .Summary }} 内容摘要11
    • 默认自动从文章头截取70个单词的内容
    • 要正确截取 CJK 语言字词, 需要在配置文件中设置 hasCJKLanguage = true
    • 手动控制截取停止位置, 只需要在正文中插入标签 <!--more-->

内容页面 Single12

文章页模板调用规则: Regular Pages

修改 single.html , 基本提供正文就行

{{ define "main" }}
    <h1>{{ .Title }}</h1>
    {{ .Content }}
{{ end }}

这样就完成了最小可用的站点功能, 运行站点可以浏览所有文章
