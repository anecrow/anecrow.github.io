用简单的 HTML+CSS 定制自己的主题.

草稿

在上一章制作的最小系统仅仅可以说是能用
显然不太符合现在的阅读习惯, 我们需要重新规划区域

首先预计一下自己需要的功能结构和基本的样式

  • 只需要一个主体承载文章和内容信息, 提高阅读专注性
    • 容器居中, 文字行不要太长
    • header放置标题和前后文章等其他信息
    • 中间放置主要内容
    • footer放置许可证和作者信息
  • 左侧栏主导航
    • 展示当前路径的页面导航
  • 右侧栏TOC(如果有)
    • 展示当前文章跳转
    • 悬浮在右下

Flex布局系统1

IE浏览器 对 Flex布局 的支持有限, 但说真的在这个年代与其考虑古董浏览器兼容性问题, 不如劝这些用户早点换个正常人应该用的东西

Flex 是一种弹性自适应尺寸的系统, 可以指定元素成为容器
容器会控制下属的直接子元素的尺寸和布局

在主题的 static/css/ 文件夹中 新建CSS样式文件 layout.css
所有影响页面版式的控制项都放在这里

在 baseof.html 定义中这部分由 partials/head.html 实现

head.html 添加引用, 全局应用同样的页面版式

<head>
    <title>{{ .Title }}</title>
    <link rel="stylesheet" href="{{ .Site.BaseURL }}css/layout.css">
</head>

语句 {{ .Site.BaseURL }} 在部署时会被替换为小站的根目录, 如 http://localhost:1313/
而 static 内的资源会原封不动地挂载在根目录下

设置 flex系统 之前对根元素做一下初始化, 编辑 layout.css

html,
body {
    display: inline-block;
    width: 100%;
    height: 100%;
    margin: 0;
    padding: 0;
}

pre {
    overflow-x: auto;
}
  • html body
    • display: inline-block
      • 设为行内块级元素, 防止奇怪的间隙和边界
    • width, height: 100%
      • 将根元素撑开至整个页面
    • margin, padding: 0
      • 防止奇怪的间隙和边界
  • pre
    • overflow-x: auto
      • 默认样式下 pre 元素会尽可能撑开横向尺寸, 导致弹性布局出现不可预料的状况
      • 设为 auto 避免元素溢出, 对显示不全的部分提供横向滑块

响应式布局的第一步, 初始化根元素的宽高为整个窗口

然后建立相关的 flex 类选择器

.flex-container {
    display: flex;
    justify-content: center;
}

.flex-container>* {
    flex-grow: 1;
    min-width: 0;
}

对于 flex容器 , 通过特殊的 css选择器 可以选择对应的 flex子元素 进行样式设置
只需要指定对应的容器, 就能免去手动指定 flex-item 的麻烦

  • .flex-container 选择主容器
    • display: flex
      • 设置元素为弹性布局容器
    • justify-content: center
      • 子元素沿主轴居中排列
  • .flex-container>* 选择子元素
    • flex-grow: 1
      • 设置子元素尺寸增长比例, 默认为 0, 设为 1 时非空子元素会更加主动地增长尺寸
      • 对稳定布局有一定好处
    • min-width: 0
      • flex 下子元素该项默认为 auto会主动将容器撑开导致内容元素溢出

另外使用 .flex-container>*:not(:is(:first-child, :last-child)) 选择器可以只影响除了头和尾的中间子元素

接着编辑 baseof.html 建立根布局

<!DOCTYPE html>
<html>
{{ partial "head.html" . }}

<body>
    <div class="flex-container">
        <div></div>
        <div id="main-container" class="flex-container">
            <div id="content">
                {{ partial "header.html" . }}
                {{ block "main" . }}{{ end }}
                {{ partial "footer.html" . }}
            </div>
        </div>
        <div></div>
    </div>
</body>

</html>

两边的空元素 <div> 不是必要的
它们会在空间允许时展开, 空间有限时压缩为零宽度
以此获得一种动态 margin 的效果, 让自适应元素看起来更顺滑

中间不为空的内容元素会无限制伸展而影响设计布局, 还需要对它指定最大尺寸 max-width

#main-container {
    max-width: 768px;
}
#content {
    max-width: 640px;
}

main-container 内部的 content 同样需要居中, 指定为 flex-container 容器类就行

添加侧边导航栏

在 main-container 两边建立容器

<body>
    <div class="flex-container">
        <div></div>
        <div class="side-container">
            {{ partial "side_nav.html" . }}
        </div>
        <div id="main-container">
            <div id="content">
                {{ partial "header.html" . }}
                {{ block "main" . }}{{ end }}
                {{ partial "footer.html" . }}
            </div>
        </div>
        <div class="side-container">
            {{ partial "side_toc.html" . }}
        </div>
        <div></div>
    </div>
</body>

别忘了限制宽度

.side-container {
    max-width: 192px;
}

在 layouts/partials/ 中新建文件 side_nav.html

<nav>
    <h1><a href="{{.Site.BaseURL}}">Home</a></h1>
    {{ range .CurrentSection.Pages.Reverse }}
    <div>
        <a href="{{.Permalink}}">{{.Title}}</a>
    </div>
    {{ end }}
</nav>

使用 range 语句迭代 .CurrentSection.Pages 得到属于当前 section 的页面列表
由于 .Pages 返回的列表默认使用 Date 从新到旧排列 加上反序关键词 .CurrentSection.Pages.Reverse 从旧到新进行迭代

同样的, 新建文件 side_toc.html

<aside>
    {{ .TableOfContents }}
</aside>

将会生成一个以正文内子标题为名的目录列表, 详情看 Table of Contents

至此小站的总体布局算是有了雏形, 之后就是细化的工作了
