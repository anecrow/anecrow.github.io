简易安装和使用 (Windows)

安装 Hugo 的方式有很多种1, 推荐简易式安装, 或者称之为绿色安装.

安装

建立 Hugo库 文件夹

新建文件夹, 随便什么名比如: hugo_sites/

可以建立在云存储中, 比如 OneDrive

获取 Hugo

为了保持简单和干净, 这里使用预编译好的程序文件
下载Windows系统可用的 Hugo Releases

比如 hugo_0.92.0_Windows-64bit.zip

需要 Sass/SCSS 功能则需要下载有着 extended 关键词命名的增强型版本
(供一些有着复杂样式的主题使用)

原地解压:

  • hugo_sites/
    • hugo_0.92.0_Windows-64bit.zip
    • hugo.exe (核心)
    • LICENSE
    • README.md

除了 hugo.exe 文件作为程序必备的核心外, 其他都是 可以删除 的文件

使用

Hugo 没有提供可视化操作UI, 需要配合 cmd 或 shell 来执行相关动作
指令 形式如: hugo <command> [<-Flag>]

因为安装时省略了配置对应 系统环境变量 的步骤, 我们需要在调用 hugo 前提供 目标位置

  • hugo.exe 在当前文件夹中时: ./hugo.exe <command> [<-Flag>]
  • hugo.exe 在上一级文件夹中时: ../hugo.exe <command> [<-Flag>]

注意当前执行命令 所在文件夹 路径和命令中 Hugo核心 的相对位置.

.exe 后缀允许省略: ./hugo <command> [<-Flag>]

建立项目

在文件夹 hugo_sites/ 中右键
选择 在 Windows 终端 中打开
输入并执行命令

建立新站点
hugo new site <name>

起名随意, 比如: new_site

./hugo new site new_site

PS ~\hugo_sites> .\hugo new site new_site

在 Windows 中路径表示通常使用反斜杠 \, 左斜杠 / 大部分情况也可用

回车执行命令, 成功时显示

Congratulations! Your new Hugo site is created in ~\hugo_sites\new_site

目录文件结构:

  • hugo_sites/
    • new_site/
      • (一些自动生成的项目框架)
    • hugo.exe

选择主题

新项目内没有用于内容展示的 layouts 模板, 目前小站 暂时无法显示

通常直接使用现成的主题模板 Hugo Themes
选择主题并遵照步骤安装至 new_site/themes/ 文件夹内
并修改 new_site/config.toml 站点配置文件, 新添一行填入对应主题名:

theme = '<theme_name>'

运行看看效果

指定了用以渲染的 Theme 之后就能运行小站了

在项目文件夹 new_site/ 中右键
选择 在 Windows 终端 中打开

注意路径处于 小站目录

执行命令

运行当前站点
hugo server

../hugo server

页面默认运行在 1313端口 : 打开主页

但由于还没有任何文章内容, 页面只有首页可用
终端中按下 ctrl C 关闭运行

新建文章

小站目录 中执行命令

新建页面
hugo new <section>/<name>.md

后缀名 .md 将指定使用 Markdown 格式编写内容

比如我们要在类别 post 中创建一个名为 new_post 的文章

../hugo new post/new_post.md

名称支持空格, 但更推荐使用下划线 _ 代替空格
路径和文件名均 支持中文

new 命令会以 archetypes/ 下对应的模板进行内容生成
修改 archetypes/default.md 文件来决定默认文章的内容模板2

展示文章

打开 new_site/post/new_post.md

---
title: "New Post"
date: 2022-01-21T23:45:30+08:00
draft: true
---

<!-- 在这里写正文 -->

文件开头由一对 --- 包围的内容为 Front Matter
储存着一些 Hugo 可读取的元数据, 但不会被正文展示

这些内容是可选的
通常它们负责给渲染模板提供更多信息

比如 title 的值会被展示为文章和页面标签的标题

新建文章会默认被标记为 draft(草稿)
使用 hugo server 命令时加入参数 -D 包含草稿内容
../hugo server -D

以 Markdown语法 写在 Front Matter 下边的则为正文部分

---
title: "New Post"
date: 2022-01-21T23:45:30+08:00
draft: true
---
