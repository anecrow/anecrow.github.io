数据结构接口

FPrimaryAssetTypeInfo

UE_5.0/Engine/Source/Runtime/Engine/Classes/Engine/AssetManagerTypes.h
官方文档: FPrimaryAssetTypeInfo

扫描资产时提供的信息对象结构接口

  • PrimaryAssetType 主资产类型
    • FName
    • 此类型主要资产的逻辑命名
  • 资产基础类
    • TObjectPtr<UClass>
    • 此类型所有资产的基类
  • 拥有蓝图类
    • bool
    • 如加载的资产为蓝图类，则为true；如为普通UObjects，则为false
  • 为仅编辑器
    • bool
    • 如此类型仅限编辑器，则为true
  • 目录
    • TArray<FDirectoryPath>
    • 搜索此资产类型的目录
  • 特定资产
    • TArray<FSoftObjectPath>
    • 要扫描的个体资产
  • 规则 FPrimaryAssetRules

    官方文档: EPrimaryAssetCookRule

    • 此类型的默认管理规则，个体资产可被覆盖
    • 优先级
      • int32
      • 为引用资产指定管理时，更高优先级的主要资产比低优先级的资产更有优先权。如优先级相同，则两者将管理相同的次要资产。
    • 数据块ID
      • int32
      • 如设为-1之外的值，资产将特定地输入此文件块ID。默认文件块为文件块0。
    • 递归应用
      • bool
      • 如为true，此规则将递归式地应用到引用的所有次要资产，前提是其未受更高优先级的主要资产所管理。
    • 烘焙规则
      • EPrimaryAssetCookRule
        • 本页位置: EPrimaryAssetCookRule
      • 描述此资产应何时被烘焙的规则。

EPrimaryAssetCookRule

UE_5.0/Engine/Source/Runtime/Engine/Classes/Engine/AssetManagerTypes.h
官方文档: EPrimaryAssetCookRule

  • enum
    • Unknown 未知
      • 不了解此资产特有的内容。如有内容依赖于资产，则将进行烘焙。
    • NeverCook 从不烘焙
      • 在任何情况喜爱都不应对资产进行烘焙/发布。如有内容依赖于资产，将出现错误。
    • DevelopmentCook 开发烘焙
      • 如有其他内容依赖，资产将在开发中被烘焙，但不会在生产版本中被烘焙。
    • DevelopmentAlwaysCook 开发固定烘焙
      • 资产将在开发中被固定烘焙，但不应在生产版本中被烘焙。
    • AlwaysCook 固定烘焙
      • 资产将在生产和开发中被固定烘焙。
